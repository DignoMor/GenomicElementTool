# import Subcommand

The `import` subcommand converts supported external inputs into Genomic Element outputs, including annotation files (`.npy`/`.npz`) and region files when applicable.

## Usage

```bash
GenomicElementTool.py import <informat> [OPTIONS]
```

## Formats

- `list`: Import a line-based text file as one stat value per region.
- `allele_expanded_ES`: Import allele-expanded exogeneous sequences and mutation metadata.

## `list`

Import one value per line from a plain text list file and save it as `region_list` annotation.

### Usage

```bash
GenomicElementTool.py import list [OPTIONS]
```

### Required Arguments

- `--region_file_path` (str)
  - Path to the input region file (BED-like format)
  - Required: Yes

- `--region_file_type` (str)
  - Type of the input region file
  - Required: Yes

- `--inpath`, `-I` (str)
  - Path to the list file (one entry per line)
  - Required: Yes

- `--opath` (str)
  - Output annotation file path
  - Required: Yes
  - Valid output extensions: `.npy`, `.npz`

### Optional Arguments

- `--dtype` (str)
  - Data type used to parse the list values
  - Default: `str`
  - Choices: `str`, `np.int32`, `np.int64`, `np.float32`, `np.float64`

### Behavior

- The number of lines in `--inpath` must match the number of regions in `--region_file_path`.
- Output annotation name is `region_list`.
- The command raises an error if:
  - line count does not match region count, or
  - `--opath` is not `.npy` or `.npz`.

### Example

```bash
GenomicElementTool.py import list \
  --region_file_path regions.bed3 \
  --region_file_type bed3 \
  --inpath region_labels.txt \
  --dtype str \
  --opath regions.region_list.npy
```

## `allele_expanded_ES`

Import allele-expanded exogeneous sequence outputs and parse mutation-aware sequence metadata.
Supports importing annotations of type `stat`.

### Usage

```bash
GenomicElementTool.py import allele_expanded_ES [OPTIONS]
```

### Required Arguments

- `--inpath`, `-I` (str)
  - Path to allele-expanded FASTA file from `export allele_expanded_ES`
  - Required: Yes

- `--anno_oheader` (str)
  - Output prefix for imported annotation files
  - Required: Yes

### Optional Arguments

- `--stat_name` (str)
  - Name of the stat annotation track to be imported
  - Can be specified multiple times
  - Must align with `--stat_npy` and `--stat_selection_method`

- `--stat_npy` (str)
  - Path to stat annotation file (`.npy` or single-array `.npz`)
  - Can be specified once per `--stat_name` argument
  - Must align with `--stat_name` and `--stat_selection_method`

- `--stat_selection_method` (str)
  - Method to select the alt stat value.
  - Choices: `max_abs_fc`
  - Can be specified once per `--stat_name` argument
  - Must align with `--stat_name` and `--stat_npy`

### Input

Expected FASTA input generated by:

```bash
GenomicElementTool.py export allele_expanded_ES ...
```

- FASTA headers should follow `export allele_expanded_ES` conventions:
  - Reference: `chrom_start_end_ref`
  - Alternate: `chrom_start_end_<snp_position>:<ref_base>2<alt_base>`
- Sequence entries represent TRE-centered reference and alternate alleles.

### Output

- **BED3 file**: `<anno_oheader>.bed3`
  - Contains reference coordinates parsed from FASTA IDs
- **Stat files** (when `--stat_name`/`--stat_npy` are provided):
  - `<anno_oheader>.<stat_name>.ref.npy`
  - `<anno_oheader>.<stat_name>.alt.npy`
  - Alternate stat values are selected by `--stat_selection_method`

### Example

Import with one stat track:

```bash
GenomicElementTool.py import allele_expanded_ES \
  --region_file_path candidate_tres.bed3 \
  --region_file_type bed3 \
  --inpath candidate_tres.allele_expanded.fa \
  --anno_oheader candidate_tres.allele_expanded \
  --stat_name count \
  --stat_npy candidate_tres.count.npy \
  --stat_selection_method max_abs_fc
```
